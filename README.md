# 🚇 GRE Tunnel Manager (v12.5)

> **کامل‌ترین ابزار مدیریت تانل GRE برای اتصال سرورهای ایران و خارج با نهایت پایداری.**

این اسکریپت فرآیند پیچیده ساخت تانل GRE، تنظیم روتینگ، پیکربندی کارت شبکه و تنظیمات فایروال را به صورت کاملاً خودکار انجام می‌دهد. با استفاده از این ابزار، می‌توانید بین سرورهای خود ارتباط لایه ۳ (IP Tunnel) برقرار کنید.

## 🌟 ویژگی‌های کلیدی

* **🚀 نصب و راه اندازی آسان:** منوی گرافیکی و رنگی برای مدیریت راحت.
* **🛡️ پایداری بالا (Watchdog):** دارای سرویس Keepalive که اتصال را چک می‌کند و در صورت قطعی، تانل را ریستارت می‌کند.
* **🌐 پشتیبانی دوال استک:** شناسایی و استفاده خودکار از **IPv4** و **IPv6**.
* **⚡ بهینه‌سازی شبکه:** اعمال خودکار تنظیمات `sysctl` و فعال‌سازی **BBR**.
* **🔀 فورواردینگ پیشرفته:** قابلیت پورت فورواردینگ روی تانل‌های موجود (گزینه ۹).
* **🔥 تنظیمات فایروال:** باز کردن خودکار پورت‌ها در `UFW` و `Iptables`.
* **مدیریت سرویس:** امکان حذف، ویرایش و مشاهده وضعیت تانل‌ها.

## 📦 نصب و اجرا

برای نصب، دستور مربوطه را در ترمینال سرور خود (هم سرور ایران و هم خارج) اجرا کنید.

### ✅ نصب نسخه رسمی (Stable)
پیشنهاد شده برای استفاده عمومی:
```bash
bash <(curl -fsSL --ipv4 https://raw.githubusercontent.com/Sir-Adnan/GRE-Tunnel-Manager/main/gre.sh)

```

### 🧪 نصب نسخه استیبل قدیمی

این نسخه قدیمی هست، اما تست شده و جواب خودش رو پس داده :

```bash
bash <(curl -fsSL --ipv4 https://raw.githubusercontent.com/Sir-Adnan/GRE-Tunnel-Manager/main/gre-stable.sh)

```

## 📖 راهنمای منوی اصلی

پس از اجرا، گزینه‌های زیر را مشاهده خواهید کرد:

| گزینه | نام | عملکرد |
| --- | --- | --- |
| **[1]** | **Kharej Server** | تنظیم سرور خارج (خلق تانل). آی‌پی ایران را وارد کنید. |
| **[2]** | **Iran Server** | تنظیم سرور ایران (خلق تانل). آی‌پی خارج را وارد کنید. |
| **[3]** | **Delete Tunnel** | حذف کامل تانل و سرویس‌های مربوطه. |
| **[4]** | **Edit Config** | ویرایش دستی فایل سرویس (مخصوص حرفه‌ای‌ها). |
| **[5]** | **Install Shortcut** | ساخت دستور میانبر `igre` برای دسترسی سریع. |
| **[6]** | **Refresh Stats** | آپدیت کردن کش آی‌پی‌ها و وضعیت لود سرور. |
| **[7]** | **Simple GRE** | ساخت تانل سریع + پورت مپینگ (بدون روتینگ پیچیده). |
| **[9]** | **Adv. Forwarding** | **بسیار مهم:** انتقال ترافیک از سرور ایران به آی‌پی داخلی تانل. |

## 🛠 آموزش راه اندازی تانل (روش استاندارد)

برای اتصال دو سرور مراحل زیر را انجام دهید:

1. **در سرور خارج (Kharej):**
* گزینه `[1] Kharej Server` را انتخاب کنید.
* آی‌پی **سرور ایران** را وارد کنید.
* یک **Tunnel ID** (مثلاً `10`) انتخاب کنید.


2. **در سرور ایران (Iran):**
* گزینه `[2] Iran Server` را انتخاب کنید.
* آی‌پی **سرور خارج** را وارد کنید.
* همان **Tunnel ID** (مثلاً `10`) را وارد کنید.


3. **پایان:**
* اسکریپت به شما یک **Internal IPv4** (مثلاً `10.0.10.1`) می‌دهد.
* اکنون دو سرور به هم متصل هستند.

## 🔀 راهنمای فورواردینگ (گزینه ۹)

بعد از اینکه تانل ساخته شد، برای اینکه کاربران بتوانند از طریق سرور ایران به اینترنت دسترسی داشته باشند:

1. در سرور **ایران**، گزینه `[9] Adv. Forwarding` را بزنید.
2. **Local Port:** پورتی که کاربر به آن وصل می‌شود (مثلاً `443`).
3. **Dest Tunnel IP:** آی‌پی داخلی تانلِ سرور خارج (که در مرحله ساخت تانل به شما نمایش داده شد، مثلاً `10.0.10.2`).
4. **Dest Port:** پورتی که سرویس شما (مثل X-UI) در سرور خارج روی آن اجرا شده است.

## ❓ سوالات متداول

**۱. چرا تانل وصل نمی‌شود؟**

* مطمئن شوید که **Tunnel ID** در هر دو سرور یکسان است.
* بررسی کنید که فایروال دیتاسنتر (Security Group) پورت پروتکل GRE را مسدود نکرده باشد.

**۲. سرویس Watchdog چیست؟**

* این سرویس به صورت خودکار هر ۱۰ ثانیه اتصال را پینگ می‌کند. اگر ارتباط قطع شده باشد، سرویس تانل را ریستارت می‌کند تا قطعی طولانی نداشته باشید.

**۳. تفاوت Simple GRE با روش معمولی چیست؟**

* روش معمولی (گزینه ۱ و ۲) یک کارت شبکه مجازی کامل می‌سازد. روش Simple (گزینه ۷) برای کسانی است که فقط می‌خواهند سریعاً یک پورت را جابجا کنند و نیاز به روتینگ پیچیده ندارند.
